# План интеграции Frontend и Backend (Signal Bot)

Дизайн перенесен на 100%, и теперь фронтенд выглядит как полноценный продукт. Ниже представлен пошаговый план работ с актуальным статусом.

---

## Этап 1: Доработка Backend API для новых UI фич ✅ ЗАВЕРШЁН

1. **Telegram Settings API (`/settings/telegram`)** ✅
   - `SettingsController` создан, сохраняет `bot_token` и `chat_id` в БД.
   - `POST /settings/telegram/verify` — отправляет тестовое сообщение и при успехе сохраняет настройки.
   - `POST /settings/telegram/test` — отправляет полноценный тестовый сигнал в подключённый канал.
   - `POST /settings/telegram/disconnect` — отключает канал.
2. **Удаление стратегий (`DELETE /strategies/:id`)** ✅
3. **Переключение статуса (`PATCH /strategies/:id/toggle`)** ✅
4. **Статус системы (`GET /health`)** ✅ — Возвращает статус db, redis, binanceWs, telegram.
5. **Тестирование сигналов (`POST /signals/test`)** ✅ — Ручная инъекция сигналов для UI-тестирования.

## Этап 2: Интеграция "Истории сигналов" (Signals) ✅ ЗАВЕРШЁН

1. **Реальные данные в Feed** ✅ — `SignalHistory` подключена к `signalsApi.getHistory()`.
2. **Live-обновления (WebSockets)** ✅ — `SignalsWsService` + `SignalsGateway` пушат `NEW_SIGNAL` в реальном времени.

## Этап 2.5: Панель свойств нод (Properties Panel) ✅ ЗАВЕРШЁН

1. **PropertiesPanel** ✅ — Полнофункциональная панель справа в Конструкторе стратегий.
   - Индикаторы: RSI (период), SMA/EMA (период), MACD (fast/slow/signal), Bollinger Bands (период + stdDev), Stochastic (K/D/smooth), ATR, Divergence (источник), Volume.
   - Таймфрейм на уровне модуля (MTF-анализ).
   - SMC-концепты: FVG, Order Block.
   - Логика: AND / OR.
   - Сигнал: LONG / SHORT.
   - **Comparison (Сравнение)** ✅ — Оператор (`>`, `<`, `==`, `cross_above`, `cross_below`) + числовое значение B.
   - Документация и визуальные гайды для каждой ноды.
2. **AST-компилятор** ✅ — Обновлён для корректного парсинга `comparison` нод с handle-ами `a`/`b` и статическим `data.value`.

## Этап 3: Интеграция "Бэктеста" (Backtest) ✅ ЗАВЕРШЁН

1. **Динамический Backtest** ✅
   - Фронтенд отправляет `POST /backtest/:strategyId` с параметрами (даты, баланс, комиссия).
   - Бэкенд возвращает список сделок с PnL.
   - **Кривая доходности (Equity Curve)** ✅ — Строится по реальным результатам сделок.
   - **Таблица сделок** ✅ — Время входа, тип, цена, PnL ($ и %).
2. **Динамическая статистика** ✅
   - Win Rate, итоговый баланс, средняя прибыль/убыток, максимальная просадка.
   - Все метрики рассчитываются из реальных данных бэкенда.
3. **Настраиваемые параметры** ✅
   - Выбор стратегии из БД, период (start/end), начальный баланс, комиссия.

## Этап 4: Интеграция "Настроек" (Settings) ✅ ЗАВЕРШЁН

1. **Проверка подключения Telegram** ✅ — Ввод bot_token + chat_id → проверка → сохранение.
2. **Дедупликация** ✅ — Ползунок (1–24 часа) с сохранением в БД.
3. **Тестовое сообщение** ✅ — Кнопка отправки полного тестового сигнала.
4. **Отключение канала** ✅.

## Этап 5: Страница "Dashboard" (Главная) ✅ ЗАВЕРШЁН

1. **Общая статистика** ✅
   - Создан `DashboardModule` (NestJS) с `GET /dashboard/stats`.
   - Агрегация: сигналы за сегодня (total/long/short/lastHour), за 7 дней, распределение по парам, по стратегиям.
   - Bar-chart данные (сигналы по дням за 7 дней) генерируются на бэкенде через `DATE_TRUNC`.
   - Hero-карточки, sidebar-статистика, прогресс-бары — всё подключено к реальным данным.
2. **Live-график цены** ✅
   - BTC/USDT 1h свечи загружаются через `candlesApi.getLatest()` (24 последних свечи).
   - Автообновление каждые 60 секунд.
3. **Динамические фильтры** ✅
   - Выпадающие списки пар и стратегий заполняются из реальных данных сигналов.
   - Индикаторы в feed отображаются из `signal.metadata.indicators`.

## Этап 6: Продвинутый бэктест + Скринер ✅ ЗАВЕРШЁН

1. **Strategy Performance Report** ✅
   - Расширенные метрики: Profit Factor, Sharpe Ratio, Max Consecutive Losses, Avg Winner vs Loser, Risk-Reward Ratio.
   - Текстовые AI-рекомендации по улучшению стратегии на основе метрик.
   - Отображение в UI: секция "Отчёт" на странице Бэктест.
2. **Multi-Symbol Screener на Dashboard** ✅
   - Виджет с live-данными: цена, 24h%, RSI, Volume для всех отслеживаемых пар.
   - Бэкенд: `GET /dashboard/screener` — агрегация текущих цен и индикаторов.
3. **WebSocket live-обновления на Dashboard** ✅
   - Подключить `useSignalsWs` для мгновенного обновления фида при поступлении нового сигнала.

## Этап 7: Оптимизация + Production ✅ ЗАВЕРШЁН

## 🚀 Backlog / План на будущее (Next Steps)

Несмотря на 100% готовность MVP, мы зафиксировали несколько крутых архитектурных улучшений для следующих версий:

1. **Динамический сканер (Топ-50 Относительный Объем)**
   - *Проблема:* Использование статических порогов (например, `Объем > 5,000,000`) не работает при смене рыночных фаз (Bull/Bear market).
   - *Решение:* Сделать условие сканера динамическим. Рассчитывать средний объем торгов по Топ-50 монетам на Binance и позволять пользователю задавать условие `Объем > 1.5x от среднего по рынку`. Это сделает сканер адаптивным к любой рыночной температуре.

2. **Продвинутая логическая валидация AST (Z3 Solver)**
   - *Проблема:* Сейчас валидатор проверяет только структурную целостность графа (отсутствие нод, пустые входы, циклы).
   - *Решение:* Интегрировать решатель для проверки взаимоисключающих условий (например, `RSI > 70 AND RSI < 30`), чтобы предупреждать трейдера о невозможных сценариях до запуска стратегии.

3. **Автоматическое тестирование сгенерированных ботов**
   - Добавить песочницу (Sandbox), которая будет компилировать сгенерированный ZIP-архив и запускать `bot.py` на тестовом датасете перед отдачей файла пользователю, чтобы гарантировать 100% валидность Python-кода.

4. **Web-интерфейс для экспортированных ботов** ✅ ЗАВЕРШЕНО
   - *Реализовано:* В генерируемый архив добавлен FastAPI сервер (`dashboard.py`) и менеджер процессов (`start.py`). Теперь пользователь может вводить ключи через браузер на `localhost:8080`.

5. **Глобальный скринер и фильтрация (Full Auto-Trading)**
   - *Задача:* Перевести сгенерированных ботов со статического списка пар на динамическое прослушивание всей биржи (`!ticker`).
   - *Интеграция:* Добавить ноды "Asset ML Statistics" в качестве математических фильтров (отменяющих сделку при плохом историческом винрейте на конкретной монете) для полностью автономного алготрейдинга.

## Архитектура проекта (справка)

### Backend (NestJS)
- **Database**: PostgreSQL + TimescaleDB
- **Cache/Queue**: Redis + Bull
- **Modules**: Health, Candles, Indicators, Strategies, Signals, Telegram, Backtest, Settings
- **WebSockets**: SignalsGateway для real-time пуша сигналов
- **Key files**:
  - `app.module.ts` — корневой модуль
  - `strategies/ast-compiler.service.ts` — компиляция React Flow графа → AST
  - `backtest/backtest.service.ts` — движок бэктеста
  - `telegram/telegram.service.ts` — отправка сигналов в Telegram
  - `settings/settings.controller.ts` — API настроек

### Frontend (React + Vite)
- **Страницы**: Dashboard, StrategyBuilder, Strategies, SignalHistory, Backtest, Settings
- **API-клиенты**: `api/strategies.ts`, `api/settings.ts`, `api/dashboard.ts`, `api/signals.ts`
- **Key components**:
  - `PropertiesPanel.tsx` — панель свойств нод (полностью функциональна)
  - `nodes/` — IndicatorNode, ComparisonNode, LogicNode, SignalNode, SmcNode и др.
  - `Layout.tsx` — общий макет с боковым меню

### Docker
- `docker-compose.yml`: postgres, redis, backend, frontend (nginx)
- Frontend собирается через Vite → production-бандл → Nginx
- Backend собирается через `nest build` → Node.js Alpine
