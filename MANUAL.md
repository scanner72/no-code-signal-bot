## 2. Начало работы и Установка

Система Signal Bot поставляется в виде Docker-контейнеров для быстрой развертки.

### Требования
- Docker & Docker Compose
- Минимум 8GB RAM (для работы моделей Kronos/Hermes)
- API Ключи Binance (режим Read-Only)

### Быстрый старт (Self-Hosted)
1. Склонируйте репозиторий.
2. Создайте файл `.env` на основе `.env.example`.
3. Запустите систему:
   ```bash
   docker-compose up -d
   ```
4. Откройте `http://localhost:5173`.
5. Перейдите в **Настройки** и подключите Telegram бота.

## 3. Главный экран (Dashboard / Control Center)

Дашборд — это ваш командный центр стратегий (Control Center), спроектированный для комплексного мониторинга работы конструктора в реальном времени. Вместо громоздких графиков и стаканов, дублирующих торговые терминалы, здесь собраны ключевые показатели эффективности, активность сигналов и управление интеграциями.

### Ключевые виджеты:
- **Панель KPI**:
  - **Активные стратегии** (общее число запущенных алгоритмов).
  - **Сигналы сегодня** (распределение по типам LONG/SHORT).
  - **Сигналы за неделю** (общая статистика за последние 7 дней).
  - **Активность за последний час** (динамика генерации новых точек входа).
  - **Статус Hermes AI** (проверка активности интеллектуальных фильтров).
- **Активность сигналов (Sparkline)**: Интерактивный мини-график активности генерации сигналов за последние 7 дней, наглядно показывающий рыночную динамику.
- **Быстрые действия (Quick Actions)**: Мгновенный переход к созданию новых стратегий в Конструкторе, запуску исторических бэктестов, каталогу готовых шаблонов или к панели настроек интеграций (heym, Hermes AI, Telegram).
- **Последние сигналы (Live Signal Feed)**: Живая лента сигналов в реальном времени, подключенная через WebSocket. Показывает тип (LONG/SHORT), пару, стратегию-источник и точное время сработки.
- **Топ стратегий**: Лидерборд лучших алгоритмов по количеству сгенерированных сигналов.
- **Market Pulse**: Компактный горизонтальный тикер внизу экрана, отображающий текущие цены основных криптовалют и процентные изменения за 24 часа.

> **Совет:** Используйте раздел "Быстрые действия" для мгновенной навигации по ключевым экранам платформы.

## 4. Интерфейс конструктора

### Панель нод (слева)

Перетащите ноду на канвас. Категории:
- **Входные данные** — markPrice, fundingRate
- **Индикаторы** — RSI, EMA, SMA, MACD, BB, Stochastic, Volume
- **Сравнение** — условия и пороги
- **Логика** — AND / OR
- **SMC концепты** — Market Structure, FVG, Order Block, Liquidity Sweep
- **Специальные** — Cross, Time Filter, Pump/Dump, Power of 3, Daily Bias
- **Order Flow** — Delta, CVD, Orderbook
- **🧠 AI Прогноз** — Sentiment, ML Filter, Hermes Agent, Kronos Forecast
- **Сигнал** — LONG / SHORT

### Канвас (центр)

| Действие | Как |
|---|---|
| Переместить ноду | ЛКМ на заголовок |
| Соединить ноды | потяните за handle с выхода на вход |
| Удалить связь | выбрать стрелку → Delete |
| Удалить ноду | выбрать ноду → Delete |
| Масштаб | Ctrl + колесо мыши |
| Перемещение | Пробел + ЛКМ |

### Inline-редактирование параметров

**Кликните на ноду** — прямо в её теле раскроются интерактивные поля для редактирования параметров. Изменения применяются мгновенно через Zustand store. Правая панель свойств также доступна для углублённой настройки.

### Верхняя панель

- **Сохранить** — сохранить стратегию в БД
- **Загрузить шаблон** — библиотека готовых стратегий
- **Импорт Pine Script** — автоматически построить граф из TradingView кода
- **Пара / Таймфрейм** — инструмент для стратегии

### 3.2 INDICATOR — Технические индикаторы

#### RSI
- **Параметры:** `period` (14)
- **Возвращает:** число 0–100
- **Применение:** RSI > 50 (бычий импульс), RSI < 30 (перепроданность)

#### SMA / EMA
- **Параметры:** `period` (SMA: 50, EMA: 20/50)
- **Применение:** EMA(20) > EMA(50) → аптренд

#### MACD
- **Параметры:** `fast` (12), `slow` (26), `signal` (9)
- **Свойства:** `histogram`, `MACD`, `signal`
- **Применение:** histogram > 0 → покупатели контролируют

#### Bollinger Bands
- **Параметры:** `period` (20), `stdDev` (2)
- **Свойства:** `upper`, `middle`, `lower`

#### Stochastic
- **Параметры:** `period` (14), `signalPeriod` (3)
- **Свойства:** `k`, `d`

#### Volume
- **Параметры:** `period` — кол-во свечей для среднего
- **Применение:** фильтрация слабых движений

### 3.3 COMPARISON — Сравнение

Сравнивает два значения, возвращает `true` / `false`.

| Оператор | Значение |
|---|---|
| `>` | A больше B |
| `<` | A меньше B |
| `>=` | A больше или равно B |
| `<=` | A меньше или равно B |
| `==` | A равно B (строки: 'BULLISH', 'bearish') |

**Поле Value:** если вход B не подключён — используется статическое значение.

### 3.4 AI — Прогнозы и Агенты

#### 🧠 Hermes Agent
- **Назначение:** Автономный LLM-фильтр (на базе Nous Hermes 3 или GPT-4).
- **Режимы:**
  - `Filter`: Возвращает PASS (пропустить) или BLOCK (заблокировать сигнал).
  - `Score`: Возвращает уверенность 0.0–1.0.
- **Логика:** Агент получает текущий рыночный контекст (цена, RSI, волатильность) и ваш текстовый промпт. Он принимает решение на основе "здравого смысла" и паттернов, которые сложно описать математически.
- **Кэширование:** Позволяет экономить API-запросы, сохраняя решение на N минут для данной пары.

#### 🔮 AI Forecast (Kronos)
- **Модели:** Kronos-Base (102M), Small (24M), Mini (4M).
- **Параметры:** `predLen` (глубина прогноза в свечах), `minConfidence` (порог уверенности).
- **Свойства:** `direction` (UP/DOWN), `predicted_change` (ожидаемое % изменение).

### 3.5 LOGIC — Логика

| Оператор | Поведение |
|---|---|
| `AND` | Все входы должны быть `true` |
| `OR` | Хотя бы один вход `true` |

Входов может быть любое количество. Выход Logic → Signal или другая Logic.

---

### 3.6 SIGNAL — Сигнал

Финальная нода. Тип: `LONG` или `SHORT`.

> Каждая стратегия **обязана** заканчиваться нодой Signal.

---

### 3.7 CROSS — Пересечение

Срабатывает **только в момент** пересечения двух линий (в отличие от Comparison).

- `above` — A пересекает B снизу вверх (бычье)
- `below` — A пересекает B сверху вниз (медвежье)

---

### 3.8 TIME FILTER — Временной фильтр

Ограничивает работу стратегии по времени суток (UTC).

- `from` / `to` — окно в формате HH:MM
- Лондон: 07:00–10:00, Нью-Йорк: 13:30–16:00, Азия: 22:00–08:00

> Выход Time Filter идёт напрямую в Logic AND.

---

### 3.9 MARKET STRUCTURE — Рыночная структура

Определяет тренд по свинг-точкам (BOS / CHoCH).

- `trend` → `'bullish'` / `'bearish'` / `'ranging'`
- `lastBOS` / `lastCHoCH` — последние события структуры
- **Параметры:** `lookback` (150)

---

### 3.9 FVG — Fair Value Gap

Определяет, находится ли цена внутри незакрытого ценового разрыва.

- **Возвращает:** `true` / `false`
- **Параметры:** `lookback`

---

### 3.10 ORDER BLOCK — Ордер Блок

Зона институционального интереса (разворотный бар перед импульсом).

- **Параметры:** `lookback`, `obType` (`'BULLISH'` / `'BEARISH'`)
- **Возвращает:** `true` если цена внутри блока нужного типа

---

### 3.11 LIQUIDITY SWEEP — Метла ликвидности

Цена пробила уровень, но закрылась обратно (манипуляция).

- **Параметры:** `sweepType` — `'HIGH'` / `'LOW'` / `'ANY'`

---

### 3.12 PUMP / DUMP

Аномальное движение цены с повышенным объёмом.

- **Параметры:** `priceThreshold` (%), `volMultiplier`, `lookback`

---

### 3.13 DAILY BIAS — Дневной байас

Тренд предыдущего дня: `'BULLISH'` / `'BEARISH'` / `'NEUTRAL'`.

> Лучше использовать на таймфреймах 15m–1h.

---

### 3.14 POWER OF 3 (PO3)

ICT-концепция: Accumulation → Manipulation → Distribution.

| Фаза | Смысл |
|---|---|
| `'ACCUMULATION'` | Консолидация у дневного открытия |
| `'MANIPULATION_LOW_COMPLETED'` | Judas Swing вниз завершён → LONG |
| `'MANIPULATION_HIGH_COMPLETED'` | Judas Swing вверх завершён → SHORT |
| `'DISTRIBUTION'` | Активное движение от открытия |

---

### 3.15 ORDER FLOW — Поток ордеров

Анализирует дисбаланс покупок/продаж и накопленную дельту.

- **Метрики:** `imbalance` (перекос стакана), `cvd` (кумулятивная дельта), `delta` (дельта текущей свечи)
- **Параметры:** `metric`, `period`
- **Применение:** CVD растёт при падении цены → скрытое накопление

---

### 3.16 ORDERBOOK — Книга ордеров

Анализ стакана в реальном времени.

- **Метрики:** `imbalance` (bid/ask ratio), `wallDetect` (крупные стены)
- **Параметры:** `levels` (глубина анализа, по умолчанию 20)

---

### 3.17 MARKET SENTIMENT — Рыночный сентимент

Агрегированный индекс настроений из новостных и социальных источников.

- **Возвращает:** `score` (−1 до +1), `label` (`'BULLISH'` / `'BEARISH'` / `'NEUTRAL'`)
- **Применение:** Comparison с `score > 0.3` для фильтра бычьего настроения

---

### 3.18 ML FILTER — ML-фильтр

Предсказание вероятности успеха сигнала на основе обученной ML-модели (Kronos).

- **Параметры:** `minConfidence` (порог уверенности, 0–1), `timeframe`
- **Возвращает:** `true` если уверенность модели выше порога
- **Применение:** добавить последним фильтром в AND — пропускать только высоковероятные сигналы

---

### 3.19 HERMES AGENT — AI-агент

Подключает LLM-агент (Nous Hermes) для интеллектуальной фильтрации сигналов на основе рыночного контекста. Вся конфигурация провайдеров и моделей теперь вынесена в **Настройки -> Интеграции** (БД-ориентированная структура с возможностью горячей перезагрузки без рестарта сервера).

**Режимы работы:**
| Режим | Поведение |
|---|---|
| `filter` | Агент решает PASS / BLOCK. Возвращает `true` / `false` |
| `score` | Агент возвращает числовой score (0–1), нужен Comparison для порога |

**Параметры ноды:**
- `mode` — `filter` или `score`
- `promptTemplate` — шаблон запроса агенту (описывает контекст рынка, баланс, индикаторы)
- `cacheDuration` — кэширование ответа в минутах (снижает latency и затраты на токены)

**Поддерживаемые провайдеры (настраиваются в Settings):**
1. **Hermes Agent (Custom)**: Прямой эндпоинт (`/run`) кастомного агента, возвращающий структурированные решения.
2. **Ollama**: Локальный запуск открытых моделей (например, `nous-hermes-3`) через эндпоинт `/api/generate`.
3. **OpenAI-совместимый API**: Подключение облачных платформ (OpenAI, Groq, Together, LM Studio, vLLM) через стандартный `/v1/chat/completions` эндпоинт с поддержкой авторизации по API-ключу.

**Как работает:**
1. Движок собирает контекст: пара, таймфрейм, цена, индикаторы
2. Подставляет в `promptTemplate` и отправляет в Hermes Agent API
3. Ответ кэшируется в Redis на `cacheDuration` минут
4. При бэктесте API не вызывается — используются детерминированные моки

> **Важно:** Hermes добавляет latency (200–500мс локально, 500–3000мс cloud). Используйте кэш.

### 3.20 KRONOS FORECAST — AI-прогноз

Прогноз направления цены на основе встроенной нейронной сети (Kronos, Ollama).

- **Параметры:** `horizon` (горизонт прогноза в свечах), `minConfidence`
- **Возвращает:** `true` при достаточной уверенности прогноза

### 3.20.1 MULTI-TIMEFRAME (MTF) ANALYSIS — Мульти-таймфрейм

Нода-обертка (контейнер), позволяющая выполнять любые вложенные условия и расчеты индикаторов на альтернативном таймфрейме, отличном от базового таймфрейма стратегии. Это позволяет создавать гибридные алгоритмы (например, вход по RSI на 5m только при бычьем тренде EMA на 1H).

- **Параметры:**
  - `timeframe` — Альтернативный таймфрейм для анализа (`1m`, `5m`, `15m`, `1h`, `4h`, `1d`).
  - `mode` — Режим анализа:
    - `Trend` — Долгосрочная фильтрация тренда (например, на 1H или 4H).
    - `Signal` — Поиск точки входа на младшем таймфрейме.
    - `Confirm` — Подтверждение паттернов на средних таймфреймах.
- **Как работает:**
  - На холсте вы соединяете логическое условие (например, `Indicator > Value`) со входом ноды `MTF`. Выход `MTF` направляете в общую логику `AND/OR` или сигнальную ноду.
  - AST-компилятор автоматически сериализует вложенный граф условий как свойство `condition` объекта MTF.
  - Движок сигналов (`SignalsEngineService`) перехватывает выполнение, динамически запрашивает исторические свечи для выбранного таймфрейма `timeframe` через `CandlesService`, кэширует их в памяти контекста на время текущей итерации и выполняет вложенные проверки на этих данных.

### 3.20.2 HEYM VALIDATOR (heym_mcp) — AI Валидация сигналов

Интегрирует self-hosted AI-платформу **heym** в качестве финального валидатора торговых сигналов. Работает как интеллектуальный шлюз, который может подтвердить сделку или заблокировать её на основе глубокого анализа рыночного контекста и кастомных правил.

- **Параметры:**
  - `mode` — Режим работы:
    - `filter` — Возвращает `true` (пропустить сигнал) или `false` (заблокировать).
    - `score` — Возвращает оценку уверенности (0.0–1.0), требующую `Comparison` для сравнения.
  - `mockBacktest` — Использовать эмуляцию на бэктесте. Если `true`, на истории запросы к heym API подменяются детерминированными положительными ответами (для экономии токенов и ускорения бэктеста).
  - `additionalContext` — Дополнительные текстовые инструкции или контекст, передаваемый в MCP-воркфлоу.
- **Интеграция:**
  - Параметры подключения (`heymUrl`, `heymApiKey`, `heymWorkflowId`) настраиваются в **Настройки API -> Интеграции -> heym MCP**.
  - Имеется встроенный механизм горячей перезагрузки конфигурации без перезапуска серверов и Docker-контейнеров.

## 6. Построение стратегии шаг за шагом

### Пример: EMA Crossover LONG

Стратегия: лонг при пересечении EMA(20) выше EMA(50) + RSI > 45.

```
EMA(20) ──→ Cross [A]
EMA(50) ──→ Cross [B]    direction = above

RSI(14) ──→ Comparison [A]   operator = >   value = 45

Cross ──────→ Logic AND
Comparison ──→ Logic AND

Logic AND ──→ Signal LONG
```

### Пример: Funding Rate Squeeze LONG

```
INPUT [fundingRate] ──→ Comparison [A]   operator = <   value = -0.001
RSI(14) ──────────────→ Comparison [A]   operator = <   value = 40

Comparison 1 ──→ Logic AND
Comparison 2 ──→ Logic AND

Logic AND ──→ Signal LONG
```

### Пример: AI-фильтрованная стратегия (Hermes)

```
EMA(20) > EMA(50) ──→ Logic AND ──→ Signal LONG
RSI > 45 ──────────→ Logic AND
Hermes Agent ──────→ Logic AND    ← фильтрует сигнал через LLM
```

### Пример: Гибридная стратегия с Мульти-таймфреймом (MTF) и AI Валидатором (heym)

Эта стратегия сочетает трендовый фильтр на старшем таймфрейме (1H), сигналы перепроданности/триггеры на младшем таймфрейме (5m) и финальную AI-валидацию через платформу heym:

```
[Таймфрейм 1H (MTF)]
EMA(50) ────→ Comparison [A]  operator = >
EMA(200) ───→ Comparison [B] (Бычий тренд на 1H) ──→ [ MTF Node (1H, Trend) ] ──→ Logic AND

[Таймфрейм 5m (базовый)]
RSI(14) ────→ Comparison [A]  operator = <  value = 30 (Перепроданность на 5m) ──→ Logic AND

[Валидация]
heym Validator (mode = filter, mockBacktest = true) ─────────────────────────────→ Logic AND

Logic AND ──→ Signal LONG
```

**JSON представление (AST):**
```json

  "nodes": [
    {
      "id": "mtf_1",
      "type": "mtf",
      "timeframe": "1h",
      "mode": "Trend",
      "condition": {
        "type": "comparison",
        "operator": ">",
        "left": {
          "type": "indicator",
          "name": "EMA",
          "params": { "period": 50 }
        },
        "right": {
          "type": "indicator",
          "name": "EMA",
          "params": { "period": 200 }
        }
      }
    },
    {
      "id": "rsi_5m",
      "type": "comparison",
      "operator": "<",
      "left": {
        "type": "indicator",
        "name": "RSI",
        "params": { "period": 14 }
      },
      "right": 30
    },
    {
      "id": "heym_val",
      "type": "heym_mcp",
      "data": {
        "mode": "filter",
        "mockBacktest": true,
        "additionalContext": "Verify macro sentiment and key exchange structures before entry."
      }
    },
    {
      "id": "and_logic",
      "type": "logic",
```

### Типовые паттерны

**Тренд:**
```
EMA(fast) > EMA(slow) → AND → Signal
RSI > 50 ─────────────→ AND
```

**ICT / Smart Money:**
```
Market Structure == 'bullish' → AND → Signal
FVG [true] ────────────────────→ AND
RSI < 50 ──────────────────────→ AND
```

**Сессионная:**
```
Time Filter 07:00–10:00 → AND → Signal
EMA(20) > EMA(50) ──────→ AND
RSI > 45 ───────────────→ AND
```

**С AI-фильтром:**
```
Cross EMA above ──→ AND → Signal
RSI > 45 ─────────→ AND
Sentiment > 0.2 ──→ AND
Hermes [filter] ──→ AND
```

## 7. Бэктест

Страница **Бэктест** проверяет стратегию на исторических данных.

### Параметры запуска

| Параметр | Описание | Рекомендация |
|---|---|---|
| **Стратегия** | Выбор из сохранённых | — |
| **Период** | Даты начала и конца | Минимум 3 месяца |
| **Начальный баланс** | Условный капитал ($) | 1000 |
| **Комиссия** | Торговая комиссия | 0.001 = 0.1% |
| **Take Profit** | TP в % от цены входа | 2–5% |
| **Stop Loss** | SL в % от цены входа | 1–2% |
| **Размер позиции** | % от баланса | 80–90% |

> Данные за новые периоды загружаются с Binance при первом запуске — до 1–3 минут.

> При бэктесте **Hermes Agent не вызывается** — используются детерминированные моки (`isBacktest: true`).

### Метрики результата

| Метрика | Что означает | Хорошее значение |
|---|---|---|
| **Win Rate** | % прибыльных сделок | > 50% |
| **Profit Factor** | Прибыли / Убытки | > 1.5 |
| **Max Drawdown** | Максимальная просадка | < 20% |
| **Sharpe Ratio** | Доходность / волатильность | > 1.0 |
| **Risk-Reward Ratio** | Средний выигрыш / убыток | > 1.5 |
| **Recovery Factor** | Прибыль / Max Drawdown | > 2.0 |

### Таблица сделок

Каждая сделка: время входа, направление, цена, PnL $/%,  метаданные индикаторов (RSI, AI-решение).

## 8. Оптимизатор стратегий

Страница **Оптимизатор** автоматически подбирает лучшие параметры стратегии с помощью **генетического алгоритма**.

### Как работает

1. Задайте стратегию и период для оптимизации
2. Алгоритм генерирует популяцию случайных комбинаций параметров
3. Каждая комбинация прогоняется через бэктест
4. Лучшие комбинации отбираются и скрещиваются (mutation + crossover)
5. Через N поколений возвращается лучший найденный набор параметров

### Что оптимизируется

| Параметр ноды | Диапазон |
|---|---|
| `period`, `minutes` | 2–200 (целые числа) |
| `threshold`, `score`, `minConfidence` | 0–1 (числа с плавающей точкой) |
| `priceThreshold`, `volMultiplier` | авто-диапазон по типу |

> Параметры `period` и `minutes` всегда округляются до целых чисел.

### Применение результата

После завершения оптимизации нажмите **Применить** — параметры будут загружены в конструктор. Запустите бэктест для финальной проверки.

## 13. Импорт Pine Script

Вставьте код TradingView — конструктор автоматически построит граф нод.

### Поддерживаемые конструкции

#### Индикаторы

```pinescript
rsi14   = ta.rsi(close, 14)
ema20   = ta.ema(close, 20)
ema50   = ta.ema(close, 50)
sma200  = ta.sma(close, 200)

// MACD — деструктуризация
[macdLine, signalLine, hist] = ta.macd(close, 12, 26, 9)
// или одной переменной
macdHist = ta.macd(close, 12, 26, 9).histogram

// Bollinger Bands
[bbUpper, bbMid, bbLower] = ta.bb(close, 20, 2)

// Stochastic
kLine = ta.stoch(high, low, close, 14)

// Средний объём
avgVol = ta.sma(volume, 20)
```

#### Пересечения

```pinescript
longCondition  = ta.crossover(ema20, ema50)
shortCondition = ta.crossunder(ema20, ema50)
```

#### Сравнения — числовые и между переменными

```pinescript
rsiOk   = rsi14 > 45           // переменная > число
trendOk = ema20 > ema50         // переменная > переменная
bbSqz   = bbUpper < 105.5
macdBull = macdHist > 0
```

#### Составные условия (AND / OR)

```pinescript
longCondition = rsiOk and trendOk and macdBull
shortCondition = rsi14 < 55 or ema20 < ema50
```

#### Сигнальные точки входа

```pinescript
// Автоматически распознаются:
strategy.entry("Long",  strategy.long,  when = longCondition)
strategy.entry("Short", strategy.short, when = shortCondition)
alertcondition(longCondition, title="Long Signal")

// Или переменные с именами:
longCondition / shortCondition / longEntry / shortEntry / buySignal / sellSignal
```

### Что создаётся автоматически

| Pine Script | Нода |
|---|---|
| `ta.rsi / ta.ema / ta.sma` | `Indicator` |
| `ta.macd(...)` (деструктурированный или `.property`) | `Indicator [MACD]` с нужным свойством |
| `ta.bb(...)` (деструктурированный или `.property`) | `Indicator [BB]` |
| `ta.stoch(...)` | `Indicator [STOCHASTIC]` |
| `ta.sma(volume, N)` | `Indicator [VOLUME]` |
| `ta.crossover / ta.crossunder` | `Cross` (A и B подключены правильно) |
| `var > N` или `var > var2` | `Comparison` (B подключён если правая часть — тоже переменная) |
| `x = a and b and c` | `Logic AND` с тремя входами |
| `x = a or b` | `Logic OR` |
| `strategy.entry("Long", strategy.long, when = cond)` | `Signal [LONG]` |
| `strategy.entry("Short", strategy.short, when = cond)` | `Signal [SHORT]` |
| `alertcondition(cond, ...)` | `Signal [LONG]` |
| Переменные-паттерны: `longCondition` / `buySignal` | `Signal [LONG]` |

> Комментарии (`// ...`) вырезаются перед парсингом.

### Ограничения

- `if / else`, `for`, пользовательские функции — **не разбираются**
- Математические выражения (`ema20 + 0.5 > ema50`) — не поддерживаются
- `strategy.close()` / `strategy.exit()` — игнорируются (только входы)
- Pine Script v6 с типами (`series<float>`) — работает, типы игнорируются

После импорта **обязательно проверьте граф** — убедитесь, что все связи корректны.

## 15. Советы и типичные ошибки

### Типичные ошибки

**❌ EMA-фильтр в контртрендовых стратегиях**

Funding Rate Squeeze + EMA(20) > EMA(50): когда фандинг экстремально отрицательный — рынок падает, EMA(20) < EMA(50). Фильтр блокирует все сигналы.

*Решение:* используйте RSI < 40 вместо EMA.

**❌ Несовместимые условия**

RSI < 35 AND MACD histogram > 0 → эти условия редко выполняются одновременно → 0 сигналов.

*Решение:* логически проверяйте совместимость перед бэктестом.

**❌ Мем-токены и Funding Rate SHORT**

Высокий фандинг на PEPE/WIF/BONK = продолжение роста. Стратегия "высокий фандинг → SHORT" дала −67% на 1000PEPE за 3 месяца.

*Решение:* Funding Rate стратегии — только на стабильных монетах (MATIC, HYPE, AVAX, ATOM).

**❌ Слишком много условий AND**

5+ условий → почти нет сигналов → бэктест неинформативен.

*Решение:* начните с 2–3 условий, добавляйте фильтры постепенно.

**❌ Hermes без кэша на высоком таймфрейме**

Hermes отправляет запрос на каждую свечу → высокий latency и расход токенов.

*Решение:* установите `cacheDuration` минимум 5–15 минут.

---

### Хорошие практики

**✅ Тестируйте на 3+ месяцах**

Меньше — возможна подгонка под один рыночный режим.

**✅ Проверяйте разные периоды**

Бычий (2024 Q1), медвежий (2022), боковик (2023 H1).

**✅ Cross для точных входов, Comparison для фильтрации**

Cross = "войти в момент события". Comparison = "фильтровать по состоянию".

**✅ Реалистичные пороги Funding Rate**

| Пара | Диапазон |
|---|---|
| BTCUSDT | −0.008% до +0.072% |
| ETHUSDT | −0.008% до +0.058% |
| ATOMUSDT | −0.45% до +0.01% |
| MATICUSDT | −0.037% до +0.45% |
| 1000PEPEUSDT | −0.12% до +0.24% |

Для BTC порог `-0.0005` никогда не срабатывает. Используйте `-0.001` и только на альткоинах.

**✅ TP/SL соотношение ≥ 1.5**

TP = 3%, SL = 1.5% → достаточно 40% WR чтобы быть в плюсе.

**✅ Используйте Оптимизатор после первого бэктеста**

Запустите бэктест с дефолтными параметрами → затем оптимизатор → сравните результаты.

**✅ Hermes в режиме score + Comparison**

Вместо `filter` используйте `score` → подключите к Comparison (`> 0.65`) → более гибкий порог.

Раздел **Настройки** критически важен для безопасной работы.

### Telegram и Уведомления

### Интеграции сторонних AI-сервисов (SaaS / Self-Hosted Ready)
Все внешние AI-сервисы настраиваются через вкладку **Интеграции** в панели настроек. Параметры хранятся в PostgreSQL (с маскированием чувствительных API-ключей) и обновляются «на лету» во всех фоновых процессах (Hot-Reload через EventEmitter2) без необходимости перезагрузки Docker-контейнеров.

- **Интеграция Heym MCP**:
  - Предназначена для запуска аналитических MCP-воркфлоу валидации сигналов.
  - Требует URL-адрес хоста Heym.
  - Встроенная кнопка **"Проверить соединение"** отправляет тестовый пинг на сервер Heym.
- **Интеграция Hermes AI Agent**:
  - Позволяет переключать провайдеров LLM: **Custom Agent** (эндпоинт `/run`), **Ollama** (локальный эндпоинт `/api/generate`), или **OpenAI-compatible** (стандартный `/v1/chat/completions` для Groq, Together, LM Studio и др.).
  - Поля настройки: URL-хост, точное имя модели и API-ключ (маскируется).
  - Кнопка **"Тест Hermes"** проверяет работоспособность выбранной модели.

### Глобальные Фильтры (Safety First)
- **Global Pause**: Мгновенно блокирует генерацию новых сигналов всеми стратегиями.
- **BTC/ETH Drop Threshold**: Если BTC упадет более чем на X% за 24ч, бот уйдет в режим ожидания (защита от рыночного обвала).

### Лимиты Риска
- **Daily Loss Limit**: Лимит убытка в $ на текущий день. При достижении — торговля останавливается.
- **Max Active Signals**: Ограничение количества одновременно открытых позиций во избежание перегрузки маржи.
- **Дедупликация**: Настройка "интервала тишины" (например, 4 часа) для одной пары, чтобы не заходить в сделку повторно на том же движении.
